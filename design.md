# Booxin 手机版联机大厅 - 设计文档

## 应用概述

Booxin 是一款我的世界联机大厅 App，用户可以在此注册/登录账号、浏览公共房间、管理好友、查看用户大厅，以及接收和处理房间邀请通知。

## 屏幕列表

| 屏幕名称 | 功能 | 是否需要认证 |
|---------|------|-----------|
| 认证屏幕（Auth Stack） | 注册/登录 | ✗ |
| 首页（Home） | 公共房间列表 | ✓ |
| 好友列表（Friends） | 好友/申请/邀请管理 | ✓ |
| 用户大厅（Lobby） | 浏览所有注册用户 | ✓ |
| 我的账户（Account） | 用户信息、修改密码、通知设置、登出 | ✓ |

## 主要内容与功能

### 1. 认证屏幕（Auth Stack）
**内容：**
- 注册表单：用户名、密码、确认密码
- 登录表单：用户名、密码
- 表单验证提示

**功能：**
- 用户注册（3-32 字符用户名，6-128 字符密码）
- 用户登录
- Token 本地存储（使用 SecureStore）
- 错误提示（400/401 错误处理）

### 2. 首页 - 公共房间列表（Home）
**内容：**
- 房间卡片列表，每张卡片显示：
  - 房间码（ABCD12）
  - 房主名称
  - 房间名（MOTD）
  - 当前玩家数 / 最大玩家数
  - 游戏版本（1.20.1）
  - 模组加载器（forge）
  - 房间状态（Active）
- 下拉刷新功能
- 空状态提示

**功能：**
- 获取公共房间列表（GET /api/rooms?isPublic=true）
- 下拉刷新列表
- 点击房间卡片查看详情（可选）
- 自动定时刷新（30 秒一次）

### 3. 好友列表（Friends）
**内容：**
- 三个标签页：
  1. **好友列表** - 显示在线状态、是否在房间、房间码
  2. **好友申请** - 待同意的申请（显示申请者名称、创建时间）
  3. **房间邀请** - 待处理的房间邀请（显示邀请者、房间码、模组信息）

**功能：**
- 获取好友面板数据（GET /api/friends）
- 同意/拒绝好友申请
- 处理（忽略）房间邀请
- 删除好友
- 搜索好友（可选快速搜索框）

### 4. 用户大厅（Lobby）
**内容：**
- 用户列表卡片，每张卡片显示：
  - 用户名
  - 在线状态
  - 是否在房间（及房间码）
  - 是否已是好友
  - 是否有待处理的好友申请（发出/收入）
- 分页加载（每页 30 条）
- 搜索/筛选功能（可选）
- 仅在线用户筛选（可选）

**功能：**
- 获取用户大厅列表（GET /api/auth/users/lobby?page=1&pageSize=30）
- 加好友（POST /api/friends/request）
- 分页导航
- 刷新列表

### 5. 我的账户（Account）
**内容：**
- 用户信息卡片：用户名、账户创建时间、最后登录时间
- 修改密码表单
- 通知设置
- 登出按钮

**功能：**
- 获取当前用户信息（GET /api/auth/me）
- 修改密码（POST /api/auth/change-password）
- 配置通知方式（本地存储）
- 登出（清除 Token 和本地数据）

## 关键用户流程

### 流程 1：注册并登录
1. 用户打开 App → 进入认证屏幕
2. 点击"注册"标签页
3. 输入用户名、密码、确认密码
4. 点击"注册"按钮 → API 调用 POST /api/auth/register
5. 注册成功 → 自动登录，跳转到首页
6. 注册失败 → 显示错误提示

### 流程 2：登录
1. 用户打开 App → 进入认证屏幕
2. 输入用户名、密码
3. 点击"登录"按钮 → API 调用 POST /api/auth/login
4. 登录成功 → Token 存储到 SecureStore，跳转到首页
5. 登录失败 → 显示错误提示

### 流程 3：浏览房间并加入好友
1. 用户登录 → 进入首页，看到房间列表
2. 下拉刷新 → 重新获取房间列表
3. 点击"好友"标签 → 进入好友列表
4. 点击"用户大厅"标签 → 浏览所有用户
5. 点击用户卡片上的"加好友"按钮 → 发送好友申请
6. 对方同意 → 用户出现在好友列表中

### 流程 4：处理房间邀请
1. 用户在好友列表 → 点击"房间邀请"标签
2. 看到待处理的邀请列表
3. 点击"同意"→ 接受邀请（可选跳转到房间详情）
4. 点击"忽略"→ 删除邀请

### 流程 5：接收邀请通知
1. 用户在任何屏幕 → 收到房间邀请
2. 根据通知设置：
   - **弹窗提醒**：屏幕中央弹出通知框
   - **后台弹出**：系统通知栏显示（使用 expo-notifications）
   - **关闭通知**：无任何提示

## 导航结构

```
Root
├── Auth Stack (未登录)
│   ├── Login/Register Screen
│   └── ...
└── App Stack (已登录)
    ├── Home (公共房间列表)
    ├── Friends (好友列表)
    ├── Lobby (用户大厅)
    ├── Account (我的账户)
    └── Notification Handler (后台处理邀请通知)
```

## 通知系统设计

### 通知类型
1. **房间邀请通知** - 当好友邀请用户进房时触发

### 通知方式（用户可配置）
1. **弹窗提醒** - 在 App 前台时，显示模态弹窗
2. **后台弹出** - 使用系统通知栏（expo-notifications）
3. **关闭通知** - 禁用所有通知

### 实现方式
- 使用 `expo-notifications` 处理推送通知
- 使用 `AsyncStorage` 存储用户通知偏好设置
- 前台时使用 React Native 模态框显示邀请
- 后台时使用系统通知

## 技术栈

| 层级 | 技术 |
|------|------|
| UI 框架 | React Native + Expo |
| 样式 | NativeWind (Tailwind CSS) |
| 路由 | Expo Router |
| 状态管理 | React Context + AsyncStorage |
| API 通信 | axios + 自定义 API 层 |
| 安全存储 | expo-secure-store |
| 通知 | expo-notifications |
| 认证 | JWT (RS256) |

## 开发优先级

1. **第一阶段**：认证系统（注册/登录）+ Token 管理
2. **第二阶段**：首页房间列表 + 下拉刷新
3. **第三阶段**：好友系统（列表、申请、邀请）
4. **第四阶段**：用户大厅 + 加好友
5. **第五阶段**：通知系统 + 通知设置
6. **第六阶段**：UI 优化 + 测试 + APK 构建
